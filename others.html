Development of web and mobile application to interact with Korean Talents

Private company - For Seoul client, South Korea

2022

limited information due to non-disclosure agreement


This application is a set of a public website for any customer to request a video or audio file to a registered talent and a mobile application only accessible by the talent to review order request and to capture the requested media.


I started the project as the main project manager to work with the customer to define the application then to follow the implementation by 1 of our back-end developer and 1 front-end developer.
As some customer partner left the project, I work closely with the main client to adjust the requirements, change the UI design plan and also take the position of the back-end developer and server manager.
This project, unfortunately, got suspended at the last stage due to strategy changes.


Keywords:
Django, Python, AWS, Cloudflare, Flutter, Google Firebase, Apple, Mobile app, camera

Back-end points

The back-end was developed in Django, python frame, used postgress as database and Vimeo platform for video hosting.
The server used Google firebase for OTP generation, accepted Google / Kakao / Apple authentication, integrated iamport payment system and channel talk for customer support. The server was hosted in a private cloud, and a test version running on AWS, and used static template UI behind a cloudflare CDN.

Front-end points

It was developped in Flutter and divided in a talent app, available in Google play store and Apple store, a web application for the public part and a web application for administrative tasks such as talent registration request, user request review, talent upload review and finance monitoring.
In the final version, we added DeepAR filter into the camera view of the talent apps.
