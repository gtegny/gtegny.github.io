Development of web and mobile application to interact with Korean Talents

Private company - For Seoul client, South Korea

2022

limited information due to a non-disclosure agreement


This application is a set of a public website for any customer to request a video or audio file to a registered talent and a mobile application only accessible by the talent to review order requests and to capture the requested media.


I started the project as the main project manager, working with the customer to define the application, and then following the implementation by one of our back-end developer and one front-end developer.
As some customer partners left the project, I worked closely with the main client to adjust the requirements, changed the UI design plan and also took on the position of the back-end developer and server manager.
This project, unfortunately, got suspended at the last stage due to strategy changes.


Keywords:
Django, Python, AWS, Cloudflare, Flutter, Google Firebase, Apple, Mobile app, camera

Back-end points

The back-end was developed in Django, a python framework, used Postgres as database and Vimeo platform for video hosting.
The server used Google firebase for OTP generation, accepted Google / Kakao / Apple authentication, integrated iamport payment system and channel talk for customer support. The server was hosted in a private cloud, and a test version ran on AWS, and used static template UI behind a cloudflare CDN.

Front-end points

It was developed in Flutter and divided into a talent app, available in Google Play Store and Apple Store, a web application for the public part and a web application for administrative tasks such as talent registration request, user request review, talent upload review and finance monitoring.
In the final version, we added DeepAR filters to the camera view of the talent apps.
